## 系统概述

CodeMatrix Studio 是一个 **AI+人类协同** 的软件开发平台，遵循"AI先行，人工审核批准，AI进入下一阶段"的开发理念。覆盖完整的软件开发生命周期：

1. **PRD 需求文档设计** - AI 根据你的想法生成完整的产品需求文档
2. **UI/UX 设计** - AI 根据已批准的 PRD 设计交互流程和组件结构
3. **全栈开发** - AI 根据已批准的设计文档生成前端和后端代码
4. **测试** - AI 生成测试计划和测试用例

所有团队成员都可以在 VS Code 中协作，支持实时同步。

## 技术架构

| 层级 | 技术栈 | 说明 |
|------|--------|------|
| VS Code 扩展 | TypeScript + esbuild | 提供 IDE 集成能力 |
| Web 界面 | React + Vite + Tailwind CSS | 协作用户界面 |
| 后端服务 | Node.js + Express + TypeScript | 工作流编排、实时协作 |
| 数据持久化 | Prisma ORM + PostgreSQL | 存储用户、项目、工作流数据 |
| 缓存/会话 | Redis | 会话管理和缓存 |
| 实时协作 | Socket.IO | WebSocket 实时同步 |
| AI 集成 | Anthropic Claude API | AI 内容生成 |

## 环境要求

- Node.js 20+
- pnpm（或 npm/yarn）
- Docker 和 Docker Compose（运行 PostgreSQL 和 Redis）
- Anthropic API 密钥（用于 AI 生成）

## 快速开始

### 1. 安装依赖

```bash
pnpm install:all
```

### 2. 使用 Docker Compose 启动数据库

```bash
docker-compose -f docker-compose.dev.yml up -d
```

这会启动：
- PostgreSQL 在 `localhost:5432`
- Redis 在 `localhost:6379`

### 3. 配置环境变量

```bash
cp backend/.env.example backend/.env
```

编辑 `backend/.env`，填入你的 Anthropic API 密钥：

```env
ANTHROPIC_API_KEY=你的API密钥
DATABASE_URL="postgresql://codematrix:codematrix-password@localhost:5432/codematrix?schema=public"
REDIS_URL=redis://redis:6379/0
JWT_SECRET=你的jwt密钥
```

### 4. 初始化数据库

```bash
cd backend
pnpm prisma db push
```

### 5. 启动开发服务

```bash
# 在项目根目录
pnpm dev
```

这会同时启动：
- 后端服务: `http://localhost:3001`
- Webview UI: `http://localhost:5173`
- VS Code 扩展开发服务: `http://localhost:3000`

### 6. 在 VS Code 中开发扩展

打开项目根目录，按 F5 启动扩展开发宿主。

## 功能说明

### 👤 用户系统

- **用户注册/登录**：使用邮箱和密码注册登录
- **会话管理**：支持查看当前会话，注销单个/所有会话
- **修改密码**：支持修改密码，修改后强制重新登录
- **用户搜索**：支持按邮箱/姓名搜索用户，用于添加团队成员

### 📁 项目管理

- **创建项目**：创建新的软件开发项目
- **编辑项目**：修改项目名称和描述
- **删除项目**：删除项目（仅所有者）
- **项目列表**：查看你参与的所有项目
- **项目搜索**：按名称/描述搜索项目
- **归档/取消归档**：软删除项目，保留数据但不在列表显示
- **项目统计**：查看工作流、阶段、评论、团队统计信息

### 👥 团队协作

- **添加团队成员**：直接通过邮箱添加
- **移除团队成员**：移除团队成员（仅所有者）
- **修改角色**：修改成员权限角色（仅所有者）
- **查看团队**：查看所有团队成员
- **项目转让**：转让项目所有权给团队成员
- **退出项目**：团队成员可以主动退出项目

### 🔗 项目邀请

- **生成邀请链接**：所有者/编辑者可以生成邀请链接
- **设置有效期**：默认为 7 天，可自定义
- **设置角色**：可设置邀请的默认角色（VIEWER/EDITOR）
- **查看活跃邀请**：列出所有未过期的邀请
- **撤销邀请**：撤销未使用的邀请（仅所有者）
- **公开邀请信息**：无需登录即可查看邀请信息
- **接受邀请**：已登录用户点击接受即可加入项目

### ⚙️ 工作流系统

系统采用四阶段递进式工作流：

| 阶段 | 说明 |
|------|------|
| PRD_DESIGN | 产品需求文档设计 |
| UI_UX_DESIGN | 界面交互设计 |
| DEVELOPMENT | 代码开发 |
| TESTING | 测试阶段 |

每个阶段的状态：

| 状态 | 说明 |
|------|------|
| PENDING | 等待处理 |
| AI_PROCESSING | AI 正在生成内容 |
| READY_FOR_REVIEW | 等待人工审核 |
| REVISION_REQUESTED | 请求修改 |
| APPROVED | 已批准 |
| COMPLETED | 已完成 |

### 🤖 AI 功能

- AI 根据阶段目标自动生成内容
- AI 输出格式：PRD 和 UI 设计使用 Markdown，开发阶段输出代码
- 支持人工编辑 AI 生成的内容
- 支持多人评论审核

### 💬 评论协作

- 在任意阶段添加评论
- 支持评论线程（回复评论）
- 支持代码位置定位（记录行列信息）

### ✅ 审批流程

- 多人审批跟踪
- 批准/请求修改反馈
- 记录批准人、批准时间

### 🔌 插件系统

- 支持动态加载插件
- 插件可以存储配置数据
- 开发模式支持热重载

### ⚡ 实时协作

基于 Socket.IO WebSocket：

- 在线用户状态（上线/离线）
- 光标位置同步
- 打字状态指示
- 内容更新实时广播
- 评论添加实时通知
- 用户通知推送

## 工作流程详解

### 创建新项目 → 启动工作流

1. 登录系统后，点击"创建新项目"
2. 输入项目名称和描述
3. 系统自动创建工作流和四个阶段（PRD → UI → 开发 → 测试）
4. AI 自动开始生成 PRD

### PRD 阶段

1. AI 生成 PRD 后，状态变为 `READY_FOR_REVIEW`
2. 团队成员可以查看内容，添加评论
3. 负责人审核后批准，状态变为 `APPROVED`
4. AI 自动进入下一阶段，开始生成 UI 设计

### UI/UX 设计阶段

1. AI 根据已批准的 PRD 生成 UI/UX 设计
2. 团队成员评审讨论
3. 批准后，AI 进入开发阶段

### 开发阶段

1. AI 根据 PRD 和 UI 设计生成完整代码
2. 开发人员可以在 AI 生成的基础上修改
3. 批准后进入测试阶段

### 测试阶段

1. AI 生成测试计划和测试用例
2. 测试人员执行测试
3. 批准后整个工作流完成

## 权限说明

系统支持三种角色：

| 角色 | 权限 |
|------|------|
| **OWNER**（所有者） | 所有权限，包括删除项目、转让所有权、归档、管理团队、管理邀请 |
| **EDITOR**（编辑） | 可以编辑项目、修改阶段内容、创建邀请、发表评论审批 |
| **VIEWER**（查看者） | 只能查看内容，发表评论，不能编辑 |

## 常见问题

### Q: 如何重置数据库？

A:
```bash
docker-compose -f docker-compose.dev.yml down -v
docker-compose -f docker-compose.dev.yml up -d
cd backend
pnpm prisma db push
```

### Q: AI 调用失败怎么办？

A: 检查：
1. `ANTHROPIC_API_KEY` 是否正确配置
2. API 密钥是否有足够额度
3. 网络是否能访问 Anthropic API

### Q: 邀请链接过期了怎么办？

A: 所有者可以重新生成一个新的邀请链接。

### Q: 我可以同时在多个设备登录吗？

A: 可以，每个登录都会创建一个新会话，可以在"会话管理"中单独注销。

### Q: 归档项目后数据会丢失吗？

A: 不会，归档只是软删除，数据保留，可以随时取消归档恢复。

### Q: 项目所有者可以退出项目吗？

A: 不可以，必须先转让所有权给其他团队成员才能退出。
